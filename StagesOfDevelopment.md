# Этапы разработки приложения "Расширенный консольный калькулятор".

## 1 этап - инициализация проекта
1. Создать репозитория проекта
2. Добавить файл .gitignore
3. Создать документ [README.md](README.md)
4. Создать документ [Требований к программному обеспечению](SoftwareRequirements.md)
5. Создать документ [Этапы разработки](StagesOfDevelopment.md)
6. Создать структуру проекта: папки include, src, data, tests
7. Создать файл с зависимостями cmake/Dependencies.cmake
8. Создать [src/main.cpp](src/main.cpp) - точку входа в приложение
9. Создать класс src/AppCalcucator [*.hpp](src/AppCalculator.hpp), [*.cpp](src/AppCalculator.cpp). Его код (*.hpp, *.cpp) в src, т.к. не предполагает повторного использования. Это не библиотека, и не требует модульного тестирования. В нем метод run() для старта приложения, этот метод должен принимать параметры командной строки, если они есть.
10. Реализовать шаблон Singleton для класса AppCalculator: static AppCalculator *instance; // статический указатель на единственный объект этого класса; и статический метод getInstance(), который вызывается на классе. Но т.к. в нем делается проверка instance - то это не может быть полем объекта, это должно быть поле, принадлежащее классу => static
11. В методе main() вызвать реализовать прием параметров командной строки и передачу их в run() класса AppCalculator, пока просто как факт передачи
12. Создать библиотеку src/Standard.cpp, include/Standard.hpp для функций стандартного калькулятора, добавив функции add(),substract(), multiply(), divide() для проверки работы приложения и тестов
13. Создать 3 файла CMakeLists.txt - в корне, в src, в tests 
14. Выполнить сборку проекта, убедиться что обе цели запускаются
14. Выполнить первый commit работающего приложения, push в репозиторий
