Git - это распределённая система контроля версий, которая позволяет отслеживать изменения в файлах, сотрудничать с другими разработчиками и управлять историей проекта.

# Репозиторий
Репозиторий (repository) - это хранилище вашего проекта вместе с историей изменений.

## Пример рабочего цикла
```bash
git status
git add .
git commit -m "feat: реализован класс Menu"
git push origin main
```
- Делайте маленькие коммиты с понятными сообщениями (feat:, fix:, docs:).
- Используйте ветки для новых функций (feature/menu, feature/calculator).
- Отмечайте ключевые этапы тегами (v0.1, v0.2, v1.0).
- Всегда проверяйте git status перед коммитом.


## Создание репозитория локально в Терминале (Командной строке), а потом заливка его на github
Вы находитесь в папке проекта (она может быть пустой или уже с кодом проекта). В ней уже создан файл README.md

```bash
# Создание нового репозитория локально (инициализация Git в текущей папке)
git init

# Добавление файла README.md в индекс (стейджинг), чтобы Git начал отслеживать его изменения
git add README.md

# Фиксация изменений локально (создание первого коммита с сообщением)
git commit -m "docs: first commit. README.md added"

# Переименование текущей ветки в main (по умолчанию может быть master)
git branch -M main

# Привязка локального репозитория к удалённому на GitHub (origin — имя удалённого репо)
git remote add origin git@github.com:ВАШ_АККАУНТ_НА_GITHUB/REPO_NAME_НА_GITHUB.git

# Отправка локальной ветки main в удалённый репозиторий origin и установка связи для будущих пушей/пулов, устанавливает «слежение» за веткой
git push -u origin main
```

## Если рекозиторий уже создан на github, и в нем уже находится хотябы 1 файл, то его надо склонировать и потом уже помещать в него код проекта
Вы находитесь в папке проекта (она может быть пустой или уже с кодом проекта). В ней уже создан файл README.md

```bash
# Клонирование существующего репозитория в текущую папку, для склонированного репозитория создается папка с именем репозитория
git clone <url>

# Клонирование существующего репозитория в текущую папку, без создания отдельной вложенной папки (текущая папка должна быть пустой, иначе возможны конфликты)
git clone <url> .
```

## Работа с файлами
```bash
#  Показать текущее состояние (какие файлы изменены, какие готовы к коммиту)
git status
# Добавить файл в индекс (подготовить к коммиту)
git add <file>
# Добавить все файлы текущего каталога, за исключением тех, которые начинаются с точки(.), будет учитываться .gitignore
git add *
# Добавить все изменённые файлы
git add .
# Удалить физически файл из текущего каталога и из индекса
git rm <file>
# Удалить файл из индекса, из репозитория, но оставить его физически. Это означает, что при фиксации файл будет удален из фиксации HEAD.
git rm --cached <file>
```

## Коммиты
Коммит (commit) - это снимок состояния проекта в определенный момент времени.

```bash
# Зафиксировать изменения с сообщением
git commit -m "feat: добавлен класс AppCalculator"
# Показать историю коммитов
git log
# Показать различия между версиями файлов
git diff
```

## Работа с удалённым репозиторием
```bash
# Отправить изменения на сервер.
git push
# Отправить ветку main и связать её с удалённой
git push -u origin main
# Получить изменения с сервера и объединить с локальными.
git pull
# Получить изменения с сервера без объединения.
git fetch
```
